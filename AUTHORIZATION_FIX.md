## 💡 为什么不能完全避免授权？

### macOS 安全限制

1. **系统级文件修改**需要 root 权限：
   - `/Library/Caches/`
   - `/var/log/`
   - `/System/Library/Caches/`

2. **系统维护操作**需要 root 权限：
   - `periodic daily/weekly/monthly`
   - 重建启动缓存
   - 刷新系统服务

### 可能的进一步优化（复杂）

❌ **不推荐**：安装 privileged helper tool
- 需要用户安装额外组件
- 增加安全风险
- 实现复杂，需要 XPC 通信

❌ **不推荐**：sudo 缓存
- `sudo -v` 只能缓存 5 分钟
- 不安全，可能被其他程序利用

✅ **当前方案最佳**：
- 预览操作无需授权
- 只在真正需要时授权
- 用户明确知道为什么需要授权

## 🐛 故障排查

**问题：预览也弹出授权**
- 检查代码是否正确设置 `elevated: false`
- 重新编译应用

**问题：正式清理不弹出授权**
- 检查 `elevated: true` 参数
- 可能已有 sudo 缓存（5分钟内）

**问题：授权后失败**
- 检查密码是否正确
- 查看右侧技术日志错误信息
- 确认 Mole CLI 已安装：`which mo`
