# Git Commit 建议

## 提交新文件

```bash
# 添加新的测试文件
git add Tests/SamplerTests.swift
git add Tests/QueryServerTests.swift
git add Tests/DiskScannerTests.swift
git add Tests/AppListParserTests.swift
git add Tests/MoleCLITests.swift

# 添加文档
git add docs/TEST_COVERAGE.md
git add docs/TESTING.md

# 添加脚本
git add scripts/run-tests.sh

# 添加项目文档
git add CLAUDE.md

# 提交
git commit -m "test: 增加 5 个 TDD 测试文件，覆盖率从 45% 提升至 70%

新增测试：
- SamplerTests (192行) - 采样器核心逻辑测试
- QueryServerTests (225行) - HTTP 服务器路由测试
- DiskScannerTests (266行) - 磁盘扫描 JSON 解析测试
- AppListParserTests (213行) - 应用列表解析器测试
- MoleCLITests (184行) - 子进程封装测试

改进：
- 测试文件数: 9 → 14 (+56%)
- 代码行数: ~800 → 1,846 (+131%)
- 覆盖关键组件: Sampler, QueryServer, MoleCLI, DiskScanner
- 添加测试运行脚本 (scripts/run-tests.sh)
- 添加详细测试覆盖报告 (docs/TEST_COVERAGE.md)

测试方法论：
- 单元测试优先，每个测试专注单一功能
- 使用临时目录和独立 DB 实例确保隔离性
- Fixture 数据避免耗时 I/O
- 条件集成测试（检查 mo 是否安装）
- 覆盖正常流程、边界条件和错误情况

运行测试:
./scripts/run-tests.sh
./scripts/run-tests.sh --coverage
"
```

## 或分多次提交

```bash
# 提交 1: 测试文件
git add Tests/*.swift
git commit -m "test: 新增 5 个核心组件的单元测试

- SamplerTests: 采样器生命周期和 DB 水合
- QueryServerTests: HTTP 路由和查询参数解析
- DiskScannerTests: JSON 解析和排序逻辑
- AppListParserTests: 应用列表合并和大小解析
- MoleCLITests: 子进程管理和超时处理

覆盖率提升 25% (45% → 70%)"

# 提交 2: 文档
git add docs/TEST_COVERAGE.md docs/TESTING.md
git commit -m "docs: 添加详细的测试覆盖报告和快速参考

- TEST_COVERAGE.md: 完整的测试统计和方法论
- TESTING.md: 测试快速参考和故障排除"

# 提交 3: 工具脚本
git add scripts/run-tests.sh
git commit -m "build: 添加测试运行脚本

支持 --coverage 和 --verbose 选项
自动检测依赖并跳过缺失的集成测试"

# 提交 4: 项目文档
git add CLAUDE.md
git commit -m "docs: 添加 CLAUDE.md 项目指南

为 Claude Code 提供完整的项目架构、约定和开发指南"
```

## 检查状态

```bash
git status
git diff --stat
```
